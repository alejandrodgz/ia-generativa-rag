## 1. Funcionalidad RAG

> **Nombre de la Funcionalidad:** `<completar por el equipo>`

Cada equipo debe identificar una funcionalidad RAG aplicable a su módulo de Evergreen (en este caso el módulo **ADM**), donde se use un LLM para realizar razonamiento o inferencia a partir de entradas y de información del dominio, generando salidas que aporten valor mediante IA Generativa.

### Contexto del módulo ADM

El módulo ADM de Evergreen gestiona:
- **Usuarios**: registro, autenticación y estados (activo/inactivo)
- **Roles y permisos**: Admin e Invitado, con control sobre páginas, opciones y módulos
- **Agrocadenas**: definición de cadenas productivas con etapas y participantes
- **Validaciones**: correo electrónico y número de teléfono

Una funcionalidad RAG útil para este módulo podría ser, por ejemplo:

> **Ejemplo orientativo** *(reemplazar con la propuesta real del equipo)*:
> Asistente inteligente de configuración de roles y permisos: dado el perfil de un nuevo usuario (cargo, módulo al que pertenece, tipo de participante en la agrocadena), el LLM razona sobre las políticas de acceso definidas en los manuales del sistema Evergreen y recomienda la configuración de permisos más adecuada, reduciendo errores de configuración manual.
